# Bitrix24: поиск дубликатов сделки при создании

Сервис слушает событие создания сделки (`ONCRMDEALADD`), берёт привязанный
контакт, ищет по телефону/email другие сделки этого же контакта и, если
находит совпадения, пишет предупреждение в комментарий (таймлайн) новой сделки.

## 1. Настройка в Битрикс24

### 1.1 Входящий вебхук (для запросов ОТ сервиса К Битрикс24)

1. Перейдите: **Настройки → Разработчикам → Другое → Входящий вебхук**.
2. Дайте права минимум на: `crm` (сделки, контакты).
3. Скопируйте URL вида:
   ```
   https://yourportal.bitrix24.ru/rest/1/xxxxxxxxxxxxxxxx/
   ```
4. Это значение — `B24_WEBHOOK_URL`.

### 1.2 Исходящий вебхук (событие ONCRMDEALADD → ваш сервер)

1. Перейдите: **Настройки → Разработчикам → Другое → Исходящий вебхук**.
2. Выберите событие: `ONCRMDEALADD`.
3. В поле "Обработчик" укажите публичный URL вашего сервера:
   ```
   https://ваш-домен.ru/webhook/deal-add
   ```
4. Сохраните — Битрикс24 покажет `application_token`. Скопируйте его в
   переменную `B24_APP_TOKEN` (опционально, но рекомендуется для защиты
   от подделки запросов).

> Важно: URL обработчика должен быть доступен из интернета по HTTPS —
> Битрикс24 не постучится на localhost или закрытый порт.

## 2. Деплой

### Вариант А — Docker Compose (рекомендуется)

```bash
cp .env.example .env
# отредактируйте .env — впишите B24_WEBHOOK_URL и B24_APP_TOKEN

docker compose up -d --build
```

Сервис поднимется на порту 3000. Дальше нужен обратный прокси
(nginx/Caddy/Traefik) с HTTPS, который проксирует внешний домен на
`localhost:3000`, например для nginx:

```nginx
server {
    listen 443 ssl;
    server_name ваш-домен.ru;

    ssl_certificate     /etc/letsencrypt/live/ваш-домен.ru/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/ваш-домен.ru/privkey.pem;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

Сертификат можно получить бесплатно через `certbot`.

### Вариант Б — просто Docker (без compose)

```bash
docker build -t b24-deal-dedup .
docker run -d --name b24-deal-dedup \
  -p 3000:3000 \
  -e B24_WEBHOOK_URL="https://yourportal.bitrix24.ru/rest/1/xxxxxxxxxxxxxxxx/" \
  -e B24_APP_TOKEN="ваш_application_token" \
  --restart unless-stopped \
  b24-deal-dedup
```

### Вариант В — без Docker, напрямую Node.js

```bash
cd app
npm install
export B24_WEBHOOK_URL="https://yourportal.bitrix24.ru/rest/1/xxxxxxxxxxxxxxxx/"
export B24_APP_TOKEN="ваш_application_token"
node server.js
```

Для постоянной работы используйте `pm2`:
```bash
npm install -g pm2
pm2 start server.js --name b24-deal-dedup
pm2 save
```

## 3. Проверка

1. Откройте `https://ваш-домен.ru/health` — должно вернуть `{"status":"ok"}`.
2. Создайте в Битрикс24 сделку с контактом, у которого телефон/email уже
   встречается в другой сделке.
3. В таймлайне новой сделки должен появиться комментарий вида:
   ```
   ⚠️ Обнаружены возможные дубликаты сделки (1):
   — Сделка #123 "Старая сделка" (стадия: NEW, создана: 2026-01-10T12:00:00+03:00)
   ```

## 4. Логика поиска дублей

- Из контакта, привязанного к новой сделке, извлекаются все телефоны и email.
- Телефоны нормализуются (только цифры, `8XXXXXXXXXX` → `7XXXXXXXXXX`).
- По каждому телефону/email ищутся другие контакты с таким же значением
  (`crm.contact.list`).
- Для каждого найденного контакта запрашиваются его сделки (`crm.deal.list`).
- Из результата исключается сама созданная сделка — остальное считается
  дубликатами.

## 5. Возможные доработки (по запросу)

- Поиск дублей ещё и по названию сделки / компании.
- Не просто комментарий, а автоматическая постановка задачи ответственному
  или уведомление в чат Битрикс24.
- Учитывать только "открытые" (не завершённые) сделки как дубликаты.
- Порог похожести названия (fuzzy match).

## 6. Безопасность

- Никогда не публикуйте `B24_WEBHOOK_URL` — он даёт полный доступ к CRM
  согласно выданным правам.
- Держите `.env` вне git (уже добавлен в `.gitignore`).
- Используйте `B24_APP_TOKEN` для проверки, что запросы на `/webhook/deal-add`
  действительно приходят от вашего портала Битрикс24.
